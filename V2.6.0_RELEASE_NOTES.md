## 🔒 安全更新

### CVE-2025-46569 漏洞响应

**漏洞信息**:
- **CVE编号**: CVE-2025-46569
- **影响版本**: OPA < v1.4.0
- **漏洞类型**: HTTP Data API 路径注入 / Rego代码注入
- **CVSS评分**: 9.1 (严重)
- **CWE分类**: CWE-94, CWE-116, CWE-20

**项目响应措施**:
1. ✅ 新增完整安全通告文档 (12.6-CVE-2025-46569安全通告.md, 692行)
2. ✅ 更新版本兼容性矩阵，强调v1.4.0+要求
3. ✅ 更新安全检查清单，新增CVE专项检查项
4. ✅ 更新所有文档的推荐版本为v1.4+

**新增安全文档内容包括**:
- 漏洞技术原理详解
- 3种攻击向量分析
- 完整的修复方案和缓解措施
- 漏洞检测脚本
- 安全加固Rego策略模板
- Kubernetes安全配置清单

## 📚 版本兼容性更新

### OPA v1.0+ 语法规范对齐

**Rego v1.0 强制语法变更**:
```rego
# ❌ OPA v0.x 旧语法（在v1.0中无效）
allow { true }
p.a { true }

# ✅ OPA v1.0+ 新语法（强制使用if/contains）
allow if { true }
p contains "a" if { true }
```

## ⚠️ 升级建议

### 立即行动项

1. **升级OPA到v1.4.0+**
   ```bash
   # 检查当前版本
   opa version
   
   # 升级到最新版本
   docker pull openpolicyagent/opa:1.4.0
   ```

2. **检查是否受CVE-2025-46569影响**
   ```bash
   # 运行检测脚本
   ./check-cve-2025-46569.sh
   ```

3. **更新Rego策略语法**
   ```bash
   # 自动转换
   opa fmt -w --rego-v1 *.rego
   
   # 手动添加import
   echo "import rego.v1" | cat - policy.rego > temp && mv temp policy.rego
   ```

4. **配置安全加固**
   - 部署system.authz授权策略
   - 配置网络隔离
   - 启用审计日志
